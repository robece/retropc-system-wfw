INSTALAR.TXT
============
Este archivo contiene información acerca de los problemas que pueden 
surgir durante la ejecución del programa Instalar Windows para 
Trabajo en Grupo, especialmente los relacionados con los programas 
residentes en memoria (TSR) o con los controladores. Si utiliza un TSR 
o si se presenta algún problema durante la instalación de Windows para 
Trabajo en Grupo, lea este documento antes de ejecutar nuevamente 
Windows para Trabajo en Grupo.

Observaciones generales sobre la instalación 
============================================
En esta sección se incluyen observaciones sobre determinadas 
configuraciones del sistema que pueden impedir la correcta instalación 
de Windows para Trabajo en Grupo. Se brinda además una solución a cada 
problema. Si está utilizando alguno de los productos de hardware o 
software mencionados, siga las recomendaciones indicadas ANTES de 
ejecutar Instalar.

Tarjetas de Red
---------------
Si utiliza alguna de las siguientes tarjetas, deberá ejecutar Instalar 
con la opción /i:

	Novell(R)/Anthem NE1500 
	Novell/Anthem NE2100 
	Advanced Micro Devices (AMD) AM2100
	Microdyne Exos(R) 105

A continuación del símbolo del sistema, escriba lo siguiente:

	instalar /i


Protectores de pantalla para MS-DOS(R)
--------------------------------------
Desactive cualquier protector de pantalla para MS-DOS antes de ejecutar 
Instalar, ya que podrían ocasionar problemas.


Computadoras Compaq(R) con MS-DOS 5.0 o superior y pantallas EGA o VGA 
----------------------------------------------------------------------
Si utiliza una versión actualizada de MS-DOS como 5.0 o superior, antes 
de instalar Windows para Trabajo en Grupo, examine el archivo CONFIG.SYS 
y asegúrese de que los controladores siguientes (en caso de estar 
presentes) aparezcan en este orden:

	HIMEM.SYS
	EMM386.EXE
	ANSI.SYS

Nota:   Si utiliza la versión MS-DOS 5.0 de Compaq, se utiliza HIMEM.EXE 
	en lugar de HIMEM.SYS y CEMM.EXE en lugar de EMM386.EXE.

Si el archivo ANSI.SYS se carga antes que HIMEM.SYS o EMM386.EXE (o si
utiliza MS-DOS 5.0 de COMPAQ antes de HIMEM.EXE o CEMM.EXE), entonces 
Instalar puede reiniciar el sistema antes de instalar Windows para 
Trabajo en Grupo. Además, si se utiliza la línea de comandos 
DOS=HIGH,UMB para cargar MS-DOS en memoria alta y EMM386.EXE se carga 
antes que ANSI.SYS, se creará la cantidad máxima de memoria alta 
disponible en una computadora Compaq.


Computadora AT&T(R) Safari
--------------------------
Si posee una computadora AT&T Safari, no podrá instalar Windows para 
Trabajo en Grupo en un directorio nuevo y mantener al mismo tiempo dos 
versiones de Windows. Deberá actualizar la versión actual, si es que 
tiene alguna. Si instala Windows para Trabajo en Grupo 3.11 en su propio 
directorio, el programa no utilizará los controladores especiales 
necesarios para ejecutarse en la computadora Safari.


Pantallas que podrán requerir controladores actualizados con Windows 
para Trabajo en Grupo
--------------------------------------------------------------------

Ciertos controladores de pantalla suministrados por otros fabricantes 
para Windows 3.0 pueden causar problemas con Windows para Trabajo en 
Grupo 3.11. El problema más serio que puede producirse al usar ciertos 
controladores de pantalla no compatibles con Windows para Trabajo en 
Grupo 3.11, es que se presente un fallo en el sistema mientras ejecuta 
el programa de instalación de Windows para Trabajo en Grupo. En este 
caso, ejecute Instalar y luego seleccione un tipo de pantalla estándar, 
como por ejemplo VGA, incluido con Windows para Trabajo en Grupo.

Con otros controladores de pantalla de Windows 3.0 puede que Windows 
para Trabajo en Grupo se instale sin problemas, aunque puede suceder 
que las fuentes TrueType no aparezcan correctamente en pantalla. 

En la Biblioteca de controladores de Windows (WDL) encontrará la versión 
actualizada del controlador, aunque también puede solicitarla al 
distribuidor o al fabricante. Para obtener más información sobre la 
Biblioteca de controladores de Windows, consulte Obtención de los 
controladores de la Biblioteca de controladores de Windows (WDL), más
adelante en este capítulo.

Uso del adaptador de vídeo ATI Mach 32
--------------------------------------
Si está utilizando un adaptador de vídeo ATI Mach 32, deberá utilizar
el controlador 8514/a incluido con Windows para Trabajo en Grupo.

Antes de ejecutar la instalación de Windows, asegúrese de que el 
adaptador Mach 32 esté instalado correctamente. En el programa Instalar 
de Mach 32, elija la opción Definir configuración. Configure el tamaĄo 
de la memoria VGA a la mitad de la RAM disponible en el adaptador Mach 
32. Dado que la mayoría de los adaptadores Mach 32 disponen 1 MB de RAM, 
por lo general el ajuste adecuado es 512K.

Uso del programa Stacker(R)
---------------------------
Si se emplea el programa Stacker para comprimir el disco duro, deben 
adoptarse ciertas medidas antes de instalar Windows para Trabajo en 
Grupo.

*   Si utiliza un archivo de intercambio con Windows para Trabajo en 
    Grupo, deberá instalarlo en el disco no comprimido. Cuando instale 
    el Stacker, puede dejar sin comprimir un mínimo de 1 MB de espacio 
    en disco para almacenar los archivos del sistema. A fin de que quede 
    espacio suficiente para el archivo de intercambio, la cantidad de 
    espacio en disco que se deje sin comprimir deberá ser por lo menos 
    el doble de la cantidad de memoria disponible en el sistema. Si el 
    sistema tiene 4 MB de memoria, deberá dejar sin comprimir al menos 8 
    MB de espacio en disco.

    Si Stacker ya está instalado en el sistema y no hay suficiente 
    espacio sin comprimir en el disco para un archivo de intercambio, 
    puede ejecutar - para aumentar el espacio - la utilidad STAC 
    incluida con Stacker. Para obtener más información, consulte la 
    documentación de Stacker.

*   Si está utilizando Stacker versión 2.0 o posterior, el programa de
    instalación de Windows para Trabajo en Grupo modificará los archivos
    CONFIG.SYS y AUTOEXEC.BAT. Cuando Instalar termine de efectuar las 
    modificaciones recibirá un mensaje al finalizar la instalación de la 
    parte de Windows, indicando qué modificaciones se han efectuado. Si 
    ha utilizado Stacker para cambiar la unidad de inicio, cuando vuelva 
    a iniciar el sistema Stacker le preguntará si desea actualizar a las 
    nuevas versiones de CONFIG.SYS y AUTOEXEC.BAT. Se recomienda hacerlo.

    Si está utilizando una versión de Stacker anterior a la 2.0 de 
    Stacker, será necesario modificar manualmente CONFIG.SYS y 
    AUTOEXEC.BAT. Para obtener información adicional, vea el tema: 
    "Modificación manual de los archivos CONFIG.SYS y AUTOEXEC.BAT" en 
    este mismo archivo.

*   Para obtener un rendimiento óptimo, no utilice el programa 
    SMARTDrive para aplicar el caché a la unidad comprimida. Hágalo 
    únicamente con la unidad no comprimida. Pueden incluirse opciones en 
    la línea SMARTDRV.EXE del archivo CONFIG.SYS para especificar a qué 
    unidades deberá aplicar el caché. 

    Por ejemplo, si la utilidad de intercambio del programa Stacker 
    intercambia las unidades C y D de tal modo que C quede como unidad 
    comprimida y D como no comprimida, en el archivo AUTOEXEC.BAT
    deberá incluir la siguiente línea:
			
	c:\windows\smartdrv.exe c- d+


Para obtener más información sobre los archivos de intercambio y la
utilización del programa SMARTDrive, consulte el capítulo 11, 
"Administración de la memoria y rendimiento", en el Manual del usuario 
de Windows para Trabajo en Grupo.


Solución de conflictos de memoria utilizando MONOUMB.386 o MONOUMB2.386
-----------------------------------------------------------------------
Si al iniciar Windows para Trabajo en Grupo se encuentra con el 
siguiente mensaje, es posible que el controlador de pantalla esté 
accediendo al rango de dirección monocromo (B000-B7FF), que impide al 
administrador de memoria utilizar este rango para los bloques de la 
memoria alta:

	"Windows no puede instalar un bloque de memoria alta 
	 en el segmento B000. Excluya este espacio de dirección 
	 utilizando la sintaxis del administrador de memoria. 
	 Para obtener más información, utilice un editor de 
	 MS-DOS para leer el archivo INSTALAR.TXT."

Para resolver este problema, intente instalar MONOUMB2.386 en el 
sistema. MONOUMB.386 es un controlador de dispositivo que permite a 
ciertos administradores de memoria utilizar el rango de dirección 
monocromo para los bloques de la memoria alta, incluso si el controlador 
de pantalla tiene acceso a este rango.

MONOUMB2.386 está incluido en la versión 3.1 de Windows para Trabajo en
Grupo, o puede solicitarlo a Microsoft. No se incluye en los disquetes
de Windows para Trabajo en Grupo 3.11. Para instalar MONOUMB2.386:

1. Copie y expanda el archivo MONOUMB2.38_ en el directorio SYSTEM de
   Windows, escribiendo lo siguiente a continuación del símbolo del
   sistema:

   expand a:\monoumb2.38_ c:\windows\system\monoumb2.386

2. Agregue la siguiente configuración a la sección [386Enh] del archivo 
   SYSTEM.INI:

   device=monoumb2.386

3. Inicie Windows para Trabajo en Grupo.

   Nota: Es posible que MONOUMB2.386 no funcione con algunos 
   administradores de memoria, como EMM386.EXE. En tal caso, puede 
   intentar utilizar MONOUMB.386, incluido en la Biblioteca de 
   controladores de Windows (WDL). Para obtener información acerca de la 
   obtención de controladores de WDL, consulte la sección "Obtención de 
   controladores de la Biblioteca de Controladores de Windows", en este 
   mismo capítulo.

También es posible excluir la región de dirección B000-B7FF, con lo que 
se especificará que el administrador de memoria no deberá intentar
utilizar este rango de memoria para los bloques de la memoria superior.
Para obtener información acerca de la exclusión de rangos de memoria
específicos, consulte el siguiente tema.


Solución de conflictos de memoria excluyendo un rango de dirección
------------------------------------------------------------------
Si al iniciar Windows para Trabajo en Grupo aparece el siguiente 
mensaje, siendo que la dirección especificada en el mismo no es B000, 
deberá excluir el rango de dirección:

	"Windows no puede instalar un bloque de memoria alta 
	 en el segmento xxxx. Excluya este espacio de dirección 
	 utilizando la sintaxis del administrador de memoria. 
	 Para obtener más información, utilice un editor de 
	 MS-DOS para leer el archivo INSTALAR.TXT."

Si la dirección especificada en el mensaje es B000, puede intentar
instalar MONOMB2.386 o MONOUMB.386, según se describe en el tema
precedente.

El método utilizado para excluir un rango de dirección depende del
administrador de memoria que esté utilizando. Por ejemplo, si está 
utilizando EMM386.EXE, es necesario eliminar la opción I=xxxx de la
línea de comando device=emm386.exe del archivo CONFIG.SYS, donde xxxx es 
el rango de dirección que comienza en la dirección especificada en el 
mensaje de error.

Si está utilizando QEMMI, es necesario incluir la opción X= en la línea 
de comando device=qemm.386.sys del archivo CONFIG.SYS. Por ejemplo, para 
excluir el rango de dirección C000-C7FF, debe especificar lo siguiente:

	device=qemm386.sys X=C000-C7FF

Para obtener más información acerca de la manera de configurar el 
archivo CONFIG.SYS, consulte la documentación de MS-DOS. Para obtener 
más información acerca de la instalación y configuración de EMM386.EXE,
consulte "Administración de la memoria y rendimiento" en el Manual del 
usuario de Windows o de Windows para Trabajo en Grupo. Para obtener más 
información acerca de la instalación y configuración de otros 
administradores de memoria, consulte la documentación adjunta al
dispositivo.


Modificación manual de los archivos CONFIG.SYS y AUTOEXEC.BAT 
--------------------------------------------------------------
Cuando Instalar ejecuta la instalación de Windows para Trabajo en Grupo, 
generalmente sitúa los archivos HIMEM.SYS, SMARTDRV.EXE y EMM386.EXE en 
el directorio de Windows para Trabajo en Grupo. Si el directorio de 
Windows para Trabajo en Grupo se encuentra en un disco no disponible al 
encender la computadora (por ejemplo, en un disco externo, 
intercambiable o no local), o si Instalar detecta algunas de las 
siguientes situaciones:

* La unidad de inicio ha sido modificada por una utilidad de compresión 
  de discos.

* El archivo CONFIG.SYS contiene referencias a controladores de
  dispositivo de bloque.

* El archivo CONFIG.SYS está definido para configuraciones múltiples.

* No hay suficiente espacio disponible en el disco duro para copiar los 
  archivos.

* Los archivos ya están instalados en el sistema, de manera que Instalar 
  no puede escribir sobre ellos (los archivos son de sólo lectura).

Instalar no modificará automáticamente los archivos CONFIG.SYS o 
AUTOEXEC.BAT, sino que guardará las modificaciones propuestas en los 
archivos CONFIG.WIN y AUTOEXEC.WIN, o en los archivos CONFIG.00x y 
AUTOEXEC.00x. En este caso, deberá mover los archivos HIMEM.SYS, 
SMARTDRV.EXE y EMM386.EXE a una unidad que esté disponible en el 
momento de encender la computadora, o bien utilizar un editor de texto 
para modificar el archivo CONFIG.SYS o AUTOEXEC.BAT, de tal modo que 
todas las referencias a HIMEM.SYS, SMARTDRV.EXE y EMM386.EXE aparezcan 
después de la línea de comandos device=, utilizada para cargar los 
controladores correspondientes a las unidades de disquete. Para obtener 
más información sobre las modificaciones propuestas, consulte los 
archivos CONFIG.WIN y AUTOEXEC.WIN, o bien CONFIG.00x y AUTOEXEC.00x.


Utilización de configuraciones de sistema múltiple con Windows para 
Trabajo en Grupo
___________________________________________________________________

La versión 6.0 o superior de MS-DOS le permite definir configuraciones
de sistema múltiple en un sólo archivo CONFIG.SYS. Cuando instale
Windows para Trabajo en Grupo 3.11, si está ejecutando MS-DOS en la
versión 6.0 o superior, Instalar examinará el archivo CONFIG.SYS en su
menú hasta hallar la sección de teclas de método abreviado y determinar
si este archivo contiene múltiple configuraciones. Si éste fuera el
caso, Instalar Windows para Trabajo en Grupo 3.11 creará un archivo
llamado CONFIG.WIN en el directorio de Windows que contenga las
entradas que debe contener el archivo CONFIG.SYS.

Instalar no modifica automáticamente al archivo CONFIG.SYS, para
preservar la integridad de la configuración del usuario en caso de que
encuentre alguna dificultad en la interpretación del entorno del mismo.
Una vez que Instalar finalice su trabajo, podrá abrir el archivo
CONFIG.SYS y el nuevo archivo CONFIG.WIN creado por Instalar,
fucionándolos.

NOTA: [Windows para Trabajo en Grupo] se usa en esta sección con el
fin de referirse al bloque de configuración del archivo CONFIG.SYS,
que se utiliza para la configuración de Windows para Trabajo en Grupo.
El nombre exacto de ese bloque puede ser distinto del usado aquí,
dependiendo del nombre que se utilizó para la configuración.

Cuando se utiliza la capacidad de configuración múltiple en MS-DOS 6 
(o con cualquier otro producto como tercera parte que provea de las 
mismas capacidades), es importante prestar atención a la siguiente 
información:

Las líneas siguientes deben figurar en cualquiera de los bloques de 
configuración, ya sea [Common] o [Windows para Trabajo en Grupo] del 
archivo CONFIG.SYS para que los componentes de la red de Windows para
Trabajo en Grupo se carguen como corresponde:

device=C:\WINDOWS\HIMEM.SYS
device=C:\WINDOWS\IFSHLP.SYS

NOTA: Si también tiene un bloque de configuración en su archivo
CONFIG.SYS para ejecutar Microsoft LAN Manager o Windows para Trabajo
en Grupo 3.1 y desea utilizarlo luego de instalar Windows para Trabajo
en Grupo 3.11, coloque la línea del controlador del dispositivo
IFSHLP.SYS en la configuración del bloque de Windows para Trabajo en
Grupo 3.11, en lugar de establecerla en el bloque de configuración de
[common]. Si coloca IFSHLP.SYS en su [Common] bloque de configuración,
preveerá que los controladores de red de LAN Manager o Windows para
Trabajo en Grupo 3.1 se carguen como corresponde.

Otras líneas de controladores de dispositivo que pueden existir tanto 
para el bloque de configuración de [Common] como para el de [Windows
para Trabajo en Grupo] incluyen:

device=C:\WINDOWS\EMM386.EXE

device=C:\WINDOWS\SMARTDRV.EXE /DOUBLE_BUFFER

Si previamente usó Windows para Trabajo en Grupo 3.1, puede eliminar
las siguientes líneas de controladores de dispositivo del bloque de
configuración para [Windows para Trabajo en Grupo] de su archivo 
CONFIG.SYS si todavía permanecen luego de haber instalado Windows para
Trabajo en Grupo 3.11:

c:\windows\protman.dos /i:c:\windows

c:\windows\<network card driver>.DOS { por ejemplo, EXP16.DOS }

c:\windows\workgrp.sys

Si estas líneas de controladores de dispositivo figuran en su archivo
CONFIG.SYS más de una vez, Instalar Windows para Trabajo en Grupo sólo
eliminará la primera vez que aparezcan. Si su bloque para [Windows para
Trabajo en Grupo] no es el primero en su archivo CONFIG.SYS, puede ser
que tenga que eliminarlos manualmente de [Windows para Trabajo en Grupo].

Los transportes correspondientes a una red de tercera parte, deberían
ser desplazados a la línea transports= line en la sección [controladores
de la red] del archivo SYSTEM.INI.

En el futuro, para cargar las tarjetas adaptadoras de red NDIS y los 
controladores de protocolos de red, no se utilizarán más los archivos
CONFIG.SYS y AUTOEXEC.BAT. Windows para Trabajo en Grupo 3.11 especifica
la configuración de la red en los archivos SYSTEM.INI y PROTOCOL.INI.

Si está utilizando diferentes controladores de tarjetas de adaptadores
de red o protocolos de red y desea conmutar entre esas diferentes
configuraciones, los archivos SYSTEM.INI y PROTOCOL.INI deben ser
cambiados para reflejar estas configuraciones. La vía más sencilla para
activar un cambio rápido de configuraciones, es la de mantener copias
separadas de los archivos SYSTEM.INI y PROTOCOL.INI que reflejen los
adaptadores o los protocolos que debe soportar.


Instalación de adaptadores de vídeo XGA en computadoras PS/2 Modelo 75
ó 90 
----------------------------------------------------------------------
Si dispone de un sistema PS/2 modelo 75 ó 90 con adaptador de vídeo 
XGA, el programa Instalar de Windows para Trabajo en Grupo configurará  
el adaptador de pantalla para una resolución de 640x480 puntos en 16 
colores. No modifique esta configuración durante la instalación. Si al 
instalar Windows para Trabajo en Grupo especifica una configuración 
diferente para XGA, puede que falle el programa de instalación.

Una vez que Windows para Trabajo en Grupo haya quedado instalado, podrá 
configurar XGA para utilizar una resolución diferente o para 256 
colores, si su monitor lo admite. Para más información, consulte 
"IBM XGA: configuración del color y la resolución" en el archivo 
LEAME.WRI. 

Si emplea la pantalla de plasma con una computadora PS/2 modelo 75, 
utilice UNICAMENTE la configuración 480x640 en 16 colores. Si intenta 
modificar esta configuración en las pantallas de plasma del modelo 75, 
regresará a MS-DOS cuando intente iniciar Windows para Trabajo en Grupo.

Configuración de una tarjeta de red IBM Token Ring de 4/16MB en una 
computadora Microchannel (MCA) 
-------------------------------------------------------------------
Si tiene una computadora MCA y está instalando el soporte de Windows 
para Trabajo en Grupo para una tarjeta de red IBM Token Ring de 4/16MB, 
asegúrese que el parámetro Maximum Transmits se configure no mayor a 
1024. Si intenta utilizar un valor más alto, la tarjeta de red no 
funcionará correctamente en su computadora.

Cómo corregir el mensaje de error "Espacio de entorno insuficiente" 
-------------------------------------------------------------------
Si al iniciar Windows para Trabajo en Grupo aparece un mensaje de 
"espacio de entorno insuficiente", existen dos métodos para resolver el 
problema:

*  Aumente el tamaĄo del entorno aĄadiendo la línea de comando SHELL= al 
   archivo CONFIG.SYS. Para más información sobre el comando SHELL y 
   sobre la modificación del archivo CONFIG.SYS, consulte el manual de 
   MS-DOS.

*  Traslade los comandos SET del archivo AUTOEXEC.BAT al principio del 
   archivo para que se ejecuten antes que los comandos que cargan 
   programas residentes en memoria (TSR) o controladores, como 
   SMARTDrive.
 
Uso de Extensiones de multimedia versión 1.0 con Windows para Trabajo en 
Grupo 
------------------------------------------------------------------------
Para utilizar las Extensiones de multimedia con Windows para Trabajo en 
Grupo, configure primeramente Windows con Extensiones de multimedia 
versión 1.0. Luego, actualice a Windows para Trabajo en Grupo 
instalándolo en el mismo directorio. Si instala Windows para Trabajo en 
Grupo antes de configurar las Extensiones de multimedia, no obtendrá 
buenos resultados de instalación con todos los archivos.

Consulte el documento en pantalla LEAME.WRI para obtener más información 
acerca de la ejecución de Extensiones de multimedia con Windows para 
Trabajo en Grupo.

Obtención de controladores de la Biblioteca de controladores de Windows 
(WDL)
-----------------------------------------------------------------------
Si posee un módem, puede cargar los archivos de la Biblioteca de 
controladores de Windows por medio de servicios de red como CompuServe, 
GEnie, Microsoft OnLine, o de varios servicios de boletines electrónicos 
para usuarios tales como el boletín electrónico de red de la Asociación
de grupos de usuarios de computadoras de red. Comuníquese con la filial 
de Microsoft más cercana (encontrará direcciones y núneros telefónicos 
en la tarjeta de Filiales internacionales). Para usar MSDL (Microsoft 
Download Service), necesitará de las siguientes configuraciones de 
comunicación:
Velocidad de transmisión en baudios
1200, 2400 o 9600
Bits de datos
8

Paridad
Ninguna
Bits de paro
1

Utilización de los programas residentes en memoria (TSR) y controladores 
con Windows para Trabajo en Grupo 
========================================================================

En la sección "Lista de programas" de este documento se mencionan 
algunos programas residentes en memoria y controladores causantes de 
problemas (tales como fallos en el sistema o información incorrecta en 
pantalla) cuando se ejecutan conjuntamente con Windows para Trabajo en 
Grupo. En algunos casos se indican métodos especiales para resolver los 
problemas.

Durante la instalación de Windows para Trabajo en Grupo, Instalar 
examina los archivos AUTOEXEC.BAT y CONFIG.SYS, y busca los programas 
residentes en memoria y controladores en ejecución. Si se detecta algún 
problema, aparecerá un mensaje en pantalla preguntando si se desea 
continuar con Instalar o si se prefiere salir del programa.

En este caso conviene que haga lo siguiente:

1.    Busque el programa TSR en la "Lista de programas" y utilice la 
      información del programa residente en memoria para determinar si 
      la ubicación detectada no afecta a la instalación de Windows para 
      Trabajo en Grupo. 

2.    Si la ubicación no afecta a la instalación, puede continuar 
      instalando Windows para Trabajo en Grupo siguiendo las 
      instrucciones de la pantalla.

      Si la ubicación afecta a Instalar e impide la instalación de 
      Windows para Trabajo en Grupo en la computadora, siga las 
      instrucciones que aparecen en pantalla para salir de Instalar y 
      efectúe los siguientes procedimientos.

3.    Salga del TSR y elimine todas las referencias al mismo de los 
      archivos AUTOEXEC.BAT y CONFIG.SYS.

4.    Vuelva a iniciar el sistema. 

5.    Vuelva a ejecutar el programa Instalar de Windows para Trabajo en 
      Grupo.

Una vez instalado correctamente Windows para Trabajo en Grupo, puede 
restaurar los programas residentes en memoria.

Instalar no detecta todos los controladores o programas TSR que se
mencionan en la "Lista de programas". Los programas TSR y los 
controladores que se detectan son los que aparecen en las secciones 
[incomptsr1] y [incomptsr2] del archivo SETUP.INF.

El programa Instalar de Windows para Trabajo en Grupo sólo modifica la 
primera sección de CONFIG.SYS y AUTOEXEC.BAT. Por lo tanto, deberá  
ejecutar Autocon después de Instalar y editar manualmente aquellas

información, consulte la documentación de Autocon.

que desee utilizar. Para obtener más información, consulte la 
documentación de BOOTCON.

Si se utiliza el programa Instalar estando cargada la versión 1.59 de
PC-Kwik, puede producirse un fallo del sistema. En la versión 2.0 de 
PC-Kwik parece haberse resuelto este problema.

Utilidad PCSXMAEM.SYS 
---------------------
Esta utilidad es incompatible con Windows para Trabajo en Grupo. Deben 
eliminarse todas las referencias a ella de los archivos CONFIG.SYS y 
AUTOEXEC.BAT antes de intentar instalar o ejecutar Windows para Trabajo 
en Grupo. La utilidad PCSXMAEM.SYS es casi idéntica a XMAEM.SYS pero  
con un nombre nuevo y con ligeras modificaciones. Se suministra con las 
versiones actuales del emulador PC/3270 de IBM. 

PCSX2EMS.SYS
------------
Windows para Trabajo en Grupo funcionará correctamente con este 
producto si se carga en una computadora que disponga de una tarjeta XMA. 
De lo contrario, este producto no es compatible con Windows para Trabajo 
en Grupo. Se suministra con el emulador PC/3270 de IBM.

PC Tools Deluxe Versión 7.1 (DATAMON)
-------------------------------------
Cuando se utiliza el programa residente DATAMON de PC Tools utilizando 
la opción \SENTRY+, puede haber problemas durante la instalación de 
Windows para Trabajo en Grupo si el espacio disponible en el disco es 
limitado. Se recomienda no cargar DATAMON con el parámetro \SENTRY+. 

Si se carga DATAMON después de la utilidad caché de disco SMARTDrive, 
puede fallar el sistema. Para obtener más información sobre el modo de 
descargar la utilidad DATAMON, consulte la documentación de PC Tools. 

PC Tools Deluxe(TM) Versión 6.0 (DESKTOP)
-----------------------------------------
Si se intenta iniciar la ejecución de PC Tools desde una sesión de MS-DOS 
dentro de Windows para Trabajo en Grupo, el sistema puede fallar o 
reiniciarse automáticamente. Si intenta utilizar cualquier comando de 
MS-DOS desde una sesión de MS-DOS dentro de Windows para Trabajo en Grupo
también puede ocurrir lo mismo. 

La versión 7.0 de PC Tools puede que resuelva este problema. Sin embargo, no 
debe ejecutarse el programa Desktop desde una sesión MS-DOS si también se ha 
cargado antes que Windows para Trabajo en Grupo, ya que podrá fallar el 
sistema al salir de Windows para Trabajo en Grupo. Conviene ejecutar Desktop 
desde su archivo PIF, que viene incluido con Windows para Trabajo en Grupo.

PC Tools Deluxe Versión 7.1 (VDEFEND) 
-------------------------------------
Es posible que no funcione correctamente el proceso de instalación de Windows 
para Trabajo en Grupo si este programa residente está cargado. VDEFEND puede 
volverse a cargar una vez instalado Windows para Trabajo en Grupo.

Utilidad caché de disco de PC Tools (PC-CACHE.COM) 
-------------------------------------------------
Esta utilidad caché de disco no debe utilizarse al mismo tiempo que 
SMARTDrive. La utilización de dos programas de caché de disco al mismo tiempo 
no aporta ninguna ventaja. Se recomienda emplear SMARTDrive, que se instala 
automáticamente con Windows para Trabajo en Grupo.

PRINT (MS-DOS)
--------------
Si imprime varios documentos extensos a través de una impresora de red y al 
mismo tiempo ejecuta el programa Instalar Windows para Trabajo en Grupo a 
través de la red, es posible que falle el sistema en el momento en que 
Instalar intente crear los grupos del Administrador de programas.

PYRO! versión 1.0
-----------------
Si se carga este protector de pantalla antes de iniciar Windows para Trabajo 
en Grupo, no será posible restaurar la pantalla de Windows para Trabajo en 
Grupo una vez que aparezca este protector de pantalla.

Administrador de memoria QMAPS Versión 5.16
-------------------------------------------
Si está utilizando el administrador de memoria QMAPS, puede que no sea 
posible configurar o iniciar Windows para Trabajo en Grupo. Al parecer, 
las nuevas versiones corregirán este problema. Para obtener una versión 
actualizada, póngase en contacto con Quadtel. Mientras tanto puede usar 
HIMEM.SYS y EMM386.EXE de Windows para Trabajo en Grupo para administrar 
la memoria.

RAMTYPE.SYS - Tarjetas AST RAMpage!(R)
--------------------------------------
El controlador RAMTYPE.SYS no es compatible con Windows para Trabajo en 
Grupo, por lo que conviene eliminar este controlador antes de instalar 
Windows para Trabajo en Grupo. Si se carga este controlador desde el 
archivo CONFIG.SYS, la línea de comando correspondiente a dicho controlador 
deberá convertirse en un comentario, para anular su efecto, del siguiente 
modo: 

	rem RAMTYPE.SYS 

Windows para Trabajo en Grupo funciona correctamente sin RAMTYPE.SYS. Sin 
embargo, otros controladores que usen RAMTYPE.SYS no se cargarán y las 
aplicaciones no podrán utilizar la memoria expandida en el sistema. 

Una vez instalado Windows para Trabajo en Grupo, podrán utilizarse las 
versiones 1.2 y posteriores de RAMTYPE.SYS durante la ejecución de Windows 
para Trabajo en Grupo. Si utiliza una versión anterior en una computadora 
de Micro Channel Architecture o en una IBM PS/2, y la tarjeta RAMpage! está  
configurada para memoria expandida y extendida, deberá actualizar el 
controlador AST RAMTYPE.SYS después de instalar Windows para Trabajo en 
Grupo. También necesitará eliminar el comando "rem" del archivo CONFIG.SYS, 
para poder cargar correctamente el controlador RAMTYPE.SYS.

Utilidad caché de disco Secretdisk II (FAST512.SYS) 
---------------------------------------------------
Esta utilidad caché de disco no debe utilizarse al mismo tiempo que 
SMARTDrive. Se recomienda utilizar SMARTDrive, que se instala 
automáticamente con Windows para Trabajo en Grupo, y eliminar todas las 
referencias a otras utilidades caché de disco de los archivos
CONFIG.SYS y AUTOEXEC.BAT. (Si Instalar detecta esta utilidad, la 
eliminará automáticamente del archivo CONFIG.SYS.)

Sidekick(R) Versiones 1.0 y 2.0, y SIDEKICK PLUS 
------------------------------------------------
Si se intenta ejecutar Sidekick como programa residente al mismo tiempo que 
Windows para Trabajo en Grupo, pueden presentarse diversos problemas. Se 
recomienda utilizar Sidekick como una aplicación para MS-DOS, utilizando un 
archivo PIF, en lugar de cargarlo como programa residente en memoria.

SoftIce Debugger
----------------
Si se carga este programa, Instalar fallará al entrar en la parte que se 
ejecuta bajo el entorno Windows para Trabajo en Grupo. Es conveniente no 
tener cargado el programa SoftIce al instalar Windows para Trabajo en Grupo.

SpeedFXR
--------
Cuando se ejecuta como utilidad residente, este programa no es compatible 
con Windows para Trabajo en Grupo. Este programa intentará "acelerar" el 
funcionamiento de los disquetes, pero lo que consigue en realidad es 
disminuir su velocidad. Si se ejecuta como SpeedFXR -X parece ser que se 
obtienen mejores resultados con Windows para Trabajo en Grupo, ya que de 
este modo el programa no queda residente en memoria y la velocidad de la 
unidad de disquete no queda afectada.

SUBST (MS-DOS)
--------------
No agregue o elimine las unidades "sustituidas" mientras se ejecuta Windows 
para Trabajo en Grupo. Este programa deberá funcionar correctamente si 
utiliza el comando SUBST antes de iniciarlo y no cambia el estado de las 
unidades sustituidas desde una sesión MS-DOS en Windows para Trabajo en Grupo. 

Utilidad caché de disco Super PC-Kwik (SUPERPCK.EXE) 
----------------------------------------------------
Esta utilidad caché de disco no debe utilizarse al mismo tiempo que 
SMARTDrive. El empleo de dos utilidades caché de disco al mismo tiempo, no 
aporta ninguna ventaja. En lugar se recomienda utilizar SMARTDrive, que se 
instala automáticamente con Windows para Trabajo en Grupo.

Controlador Trantor T100 Host Adapter (TSCSI.SYS) 
-------------------------------------------------
(Para unidades de disco duro SCSI) - Si usa el controlador Trantor T100 
(TSCSI.SYS), deberá utilizar la versión 4.33d o posterior. En las versiones 
anteriores, las unidades de disco SCSI se identificaban como unidades 
extraíbles. Esto puede ocasionar que falle el sistema al utilizar el 
Administrador de archivos para dar formato a dichas unidades, produciendo 
otros resultados imprevistos al eliminar archivos. Puede solicitar 
una versión actualizada de este controlador a Trantor Systems, Ltd. 

UMB PRO Versión 1.07
--------------------
Si se ejecuta UMB PRO, pueden presentarse diversos problemas al utilizar 
Windows para Trabajo en Grupo en modo estándar. En ciertos casos puede fallar 
el sistema. Asegúrese de que sólo se esté usando un administrador de memoria 
con Windows para Trabajo en Grupo. Se recomienda usar los administradores de 
memoria HIMEM.SYS y EMM386.EXE, que son instalados en forma automática con 
Windows para Trabajo en Grupo.

Vaccine
-------
Si está cargado Vaccine, Instalar fallará al intentar modificar los archivos 
CONFIG.SYS y AUTOEXEC.BAT. Debe descargar Vaccine antes de instalar 
Windows para Trabajo en Grupo. También pueden producirse otros problemas,
como sonidos, si ejecuta Windows para Trabajo en Grupo mientras Vaccine
está cargado.

VIREX-PC Versión 1.11
---------------------
Si se instala el programa VIREXPC.COM, Instalar y Windows para Trabajo en 
Grupo 3.1 fallarán cuando el programa KRNL386.EXE efectúe una operación de 
escritura. Además, la primera vez que se cargue Windows para Trabajo en 
Grupo estando instalado Virex-PC puede que vuelva a MS-DOS.

Virus Prevention Plus Versión 5.0
---------------------------------
Si este programa ya está cargado es posible que no pueda instalar Windows 
para Trabajo en Grupo correctamente. Además se presentarán problemas cuando 
se ejecute Windows para Trabajo en Grupo. El programa Keyboard Lock puede 
causar la falla de Windows para Trabajo en Grupo. 

Virusafe Versión 4.0
--------------------
Si este programa está instalado como utilidad residente en el momento de 
ejecutar la instalación de Windows para Trabajo en Grupo, puede que falle 
Instalar. Se recomienda cargar Virusafe con el parámetro /C-, que desactiva la 
detección de modificaciones en memoria que proporciona el programa. En la 
versión 4.5 de este producto parece haberse resuelto este problema. Para
obtener más información al respecto, consulte la documentación de Virusafe.

XMAEM.SYS
---------
XMAEM.SYS es un dispositivo que hace que la memoria extendida actúe como un 
dispositivo XMA (estándar de memoria extendida de IBM). Es incompatible con 
Windows para Trabajo en Grupo. Se recomienda su eliminación de los archivos 
CONFIG.SYS y AUTOEXEC.BAT antes de instalar o ejecutar Windows para Trabajo 
en Grupo. Esta utilidad está incluida con IBM DOS 4.0.

XMA2EMS.SYS
-----------
Windows para Trabajo en Grupo funcionará correctamente con este producto si 
está cargado en una computadora que disponga de una tarjeta XMA. En caso 
contrario, es incompatible con Windows para Trabajo en Grupo. Esta utilidad 
está incluida con IBM DOS 4.0.
